🎨 Design Specs:
   • Modal width: 500px (desktop), 95% (mobile)
   • Background: #0D0D0D (near-black)

⚙️ How It Works:
   1. User lands on homepage

  POPUP #2: ACCESS GATE (REDESIGNED)
═══════════════════════════════════════════════════════════════

✅ Features Implemented:
   • Dark card (#0D0D0D) on cream background (#F2EDE6)
   • MONTREZ logo (Playfair Display, serif)
   • "ACCESS REQUIRED" subtitle (uppercase, spaced)
   • Two access options:
     → Returning User: Password input (expandable)
     → New User: Instagram link for access code
   • "OR" divider between options
   • Password validation with error states
   • Shake animation on incorrect password
   • Auto-focus on password input
   • Cancel button to collapse input
   • Mobile responsive
   • Accessibility compliant

📁 Files Created:
   ├── src/components/AccessGate.jsx (4.8 KB)
   └── src/styles/AccessGate.css (4.8 KB)

🎨 Design Specs:
   • Card width: 480px (desktop), 95% (mobile)
   • Card BG: #0D0D0D (near-black)
   • Page BG: #F2EDE6 (cream)
   • Logo: 3rem, Playfair Display, 0.4em spacing
   • Options: #1A1A1A → #222 (hover)
   • Divider: #333 lines with centered "OR"
   • Instagram icon: #888 → #FFFFFF (hover)

⚙️ How It Works:
   Option A (Password):
   1. Click "Returning User"
   2. Input expands with auto-focus
   3. Type password → click "Enter"
   4. Validates via /api/verify-password
   5. Success → proceeds to video intro
   6. Error → shake + error message

   Option B (Instagram):
   1. Click "New User"
   2. Opens Instagram in new tab
   3. User gets code from DM/bio
   4. Returns and uses password option
